## Pricing

### Arc-Enabled Servers
- **Control plane** (inventory, tagging, policy, resource graph) — **Free**
- **Azure Monitor** — Per-GB ingestion pricing for logs; metrics retention pricing
- **Microsoft Defender for Servers** — Plan 1: ~$5/server/month; Plan 2: ~$15/server/month
- **Azure Update Manager** — Free for Arc-enabled servers (included)
- **Extended Security Updates** — Priced per core; see [ESU pricing](https://learn.microsoft.com/en-us/windows-server/get-started/extended-security-updates-overview)

### Arc-Enabled Kubernetes
- **Control plane and GitOps** — **Free** (Flux configurations, cluster connect, Azure Policy for K8s)
- **Cluster extensions** — Priced per extension (Defender for Containers, Azure Monitor, etc.)

### Arc-Enabled Data Services
- **SQL Managed Instance** — Priced per vCore/hour; General Purpose and Business Critical tiers
- **PostgreSQL** — Priced per vCore/hour
- **Pay-as-you-go** available in direct connected mode
- See [Azure Arc pricing](https://azure.microsoft.com/en-us/pricing/details/azure-arc/) for current rates

## Support

### Support Plans
- **Basic** — Included with all Azure subscriptions (billing and subscription support)
- **Developer** — $29/month (business hours, email support)
- **Standard** — $100/month (24/7 for Sev A/B, &lt;1 hour response for critical)
- **Professional Direct** — $1,000/month (24/7, advisory services, ProDirect delivery manager)
- **Unified** — Enterprise-wide support with dedicated TAM

### Filing Support Tickets
1. Navigate to **Help + Support** in the Azure portal
2. Select **Create a support request**
3. Service type: **Azure Arc** → select sub-service (servers, Kubernetes, data services)
4. Describe the issue, attach logs, and select severity

### Diagnostic Tools
```bash
# Server agent diagnostics
azcmagent show

# Kubernetes agent diagnostics

  --resource-group "rg-name"

# Collect Arc K8s agent logs
kubectl logs -n azure-arc -l app.kubernetes.io/part-of=arc
```

## Related Azure Services

### Azure Stack HCI
**Azure Stack HCI** is a hyperconverged infrastructure solution that runs on validated hardware and integrates deeply with Azure Arc. It provides:
- Azure-managed VMs on-premises
- AKS hybrid (Kubernetes on HCI)
- Azure Virtual Desktop on-premises
- Arc-enabled by default

### Azure VMware Solution
**Azure VMware Solution (AVS)** runs VMware vSphere natively in Azure. Arc integration enables:
- Projecting VMware VMs into Azure Resource Manager
- Applying Azure governance to VMware workloads
- Using Azure services (Monitor, Defender) on VMware VMs

### Azure Stack Edge
**Azure Stack Edge** is a managed edge appliance that supports:
- Running Kubernetes at the edge with Arc management
- AI/ML inferencing at the edge
- Data preprocessing and transfer to Azure

### Azure IoT Operations
**Azure IoT Operations** leverages Arc-enabled Kubernetes to deploy IoT services on edge clusters:
- MQTT broker for device communication
- Data processor pipelines
- Connector for OPC UA (industrial protocols)

## Bicep / Terraform Resources

### Bicep Template for Arc-Enabled Server Onboarding
```bicep
resource arcServer 'Microsoft.HybridCompute/machines@2024-07-10' existing = {
  name: 'web-server-01'
}

resource amaExtension 'Microsoft.HybridCompute/machines/extensions@2024-07-10' = {
  parent: arcServer
  name: 'AzureMonitorLinuxAgent'
  location: arcServer.location
  properties: {
    publisher: 'Microsoft.Azure.Monitor'
    type: 'AzureMonitorLinuxAgent'
    autoUpgradeMinorVersion: true
  }
}
```

### Terraform Provider
The `azurerm` Terraform provider supports Arc resources:
- `azurerm_arc_machine` — Arc-enabled server data source
- `azurerm_arc_kubernetes_cluster` — Arc-enabled Kubernetes cluster
- `azurerm_arc_machine_extension` — VM extensions on Arc servers

## Quick Reference: Key CLI Commands

| Action | Command |
|---|---|
| List Arc servers | `az connectedmachine list -g RG -o table` |
| Show server details | `az connectedmachine show -n NAME -g RG` |
| List K8s clusters | `az connectedk8s list -g RG -o table` |
| Connect K8s cluster | `az connectedk8s connect -n NAME -g RG` |
| Create Flux config | `az k8s-configuration flux create ...` |
| Install extension | `az k8s-extension create ...` |
| Check agent health | `azcmagent show` / `azcmagent check` |
