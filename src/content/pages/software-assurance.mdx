## Software Assurance + Azure Arc

Software Assurance (SA) is a Microsoft Volume Licensing benefit — but for Azure Arc customers, it's more than just a maintenance agreement. **SA unlocks an entire tier of Azure Arc management capabilities at no additional cost** that would otherwise require per-VM monthly fees.

If your organization has Windows Server or SQL Server licenses with active SA (through EA, EAS, MPSA, or Select Plus agreements), you're leaving money on the table if you're not using these benefits with Azure Arc.

## Windows Server Management Enabled by Azure Arc

Azure Arc has three tiers of capability for Windows Server. SA customers get access to the **premium tier** — "Windows Server management enabled by Azure Arc" — at no extra cost.

### What SA Unlocks (vs Paid)

| Capability | Without SA (Basic Arc) | With SA |
|-----------|----------------------|---------|
| Azure Change Tracking and Inventory | **$6/VM/month** | ✅ Included |
| Azure Policy Machine Guest Configuration | **$6/VM/month** (bundled with above) | ✅ Included |
| Azure Update Manager | **$5/VM/month** | ✅ Included |
| Windows Server Best Practices Assessment | ❌ Not available | ✅ Included |
| Remote Support | ❌ Not available | ✅ Included |
| Disaster Recovery | ❌ Not available | ✅ Included |
| Network HUD / Accelnet (WS 2025) | ❌ Not available | ✅ Included |

**Key detail:** Some of these capabilities aren't just discounted for SA customers — they're **exclusive**. Best Practices Assessment, Remote Support, Disaster Recovery, and Network HUD are only available through the SA-enabled tier. You can't buy them separately on Basic Arc.

### What This Means in Real Dollars

For a 100-server environment on Basic Arc without SA:
- Change Tracking + Guest Config: 100 × $6/mo = **$600/month**
- Azure Update Manager: 100 × $5/mo = **$500/month**
- **Total: $1,100/month ($13,200/year)** — just for these three capabilities

With SA, all of that is **$0/month** — plus you get the four exclusive capabilities on top.

### What's Always Free (Basic Arc)

These capabilities are available to all Arc-enabled servers regardless of SA:

- Tagging and inventory management
- Microsoft Entra ID authentication
- Role-Based Access Control (RBAC)
- Azure Resource Manager integration
- Windows Admin Center
- Azure Migrate assessments and business case

### What's Always Paid (Both Tiers)

These services are paid add-ons regardless of SA status:

- Microsoft Copilot in Azure (preview)
- Hotpatch
- Microsoft Defender for Cloud
- Azure Monitor
- Microsoft Sentinel
- Extended Security Updates

## SQL Server Management with Arc + SA

Arc-enabled SQL Server brings Azure-grade management to on-prem and third-party cloud SQL instances. SA enhances this with **ESU entitlement and pay-as-you-go flexibility**.

### Arc-Enabled SQL Server Capabilities

| Capability | Standalone SQL | Arc-Enabled SQL |
|-----------|---------------|----------------|
| Pay-as-you-go billing | ❌ | ✅ |
| Microsoft Entra ID auth | ❌ | ✅ |
| Best practices assessment | ❌ | ✅ |
| Inventory management | ❌ | ✅ |
| Automated patching | ❌ | ✅ |
| Automated backup | ❌ | ✅ |
| Performance monitoring | ❌ | ✅ |
| Microsoft Defender for SQL | ✅ | ✅ |
| HA/DR inventory management | ❌ | ✅ |
| Microsoft Purview (data governance) | ❌ | ✅ |
| Point-in-time restore | ❌ | ✅ |

Without Arc, standalone SQL Server supports **one** Azure management capability (Defender). With Arc, you get **eleven** — and SA customers get ESU coverage on top.

### SQL Server SA Benefits

- **ESU at no additional cost** — SA-covered SQL Server licenses include ESU when enrolled through Arc. Without SA, ESU is 75% of license cost per year, cumulative.
- **Azure Hybrid Benefit** — Use existing SQL Server licenses on Azure VMs, saving up to 85% vs pay-as-you-go
- **License Mobility** — Move SQL Server licenses between on-prem, third-party clouds, and Azure

## Azure Hybrid Benefit (AHB)

The most direct SA cost savings for Azure customers. AHB lets you apply existing on-prem licenses to Azure resources.

### Windows Server AHB

Each **Windows Server Standard** SA license (16 cores) covers up to **2 Azure VMs** with up to 8 vCPUs each. **Datacenter edition** covers **unlimited Azure VMs**.

| VM Size | Pay-as-you-go | With AHB | Savings |
|---------|--------------|----------|---------|
| D4s v5 (4 vCPUs) | ~$280/mo | ~$140/mo | **~50%** |
| D8s v5 (8 vCPUs) | ~$560/mo | ~$280/mo | **~50%** |
| E16s v5 (16 vCPUs) | ~$1,120/mo | ~$560/mo | **~50%** |

### SQL Server AHB

| Instance | Pay-as-you-go | With AHB | Savings |
|----------|--------------|----------|---------|
| SQL MI General Purpose (8 vCores) | ~$1,500/mo | ~$450/mo | **~70%** |
| SQL MI Business Critical (8 vCores) | ~$4,200/mo | ~$1,100/mo | **~74%** |
| SQL on Azure VM (Enterprise, 8 vCPUs) | ~$2,400/mo | ~$280/mo | **~88%** |

### Enabling AHB

```bash
# Enable AHB on an Azure VM
az vm update \
  --name "vm-prod-01" \
  --resource-group "rg-production" \
  --license-type "Windows_Server"

# Enable AHB for SQL Managed Instance
az sql mi update \
  --name "sqlmi-prod" \
  --resource-group "rg-production" \
  --license-type "BasePrice"
```

In the portal: VM → **Configuration** → **Licensing** → select **Azure Hybrid Benefit**.

## Disaster Recovery Rights

SA provides **free failover rights** — no additional licenses needed for DR replicas:

- **Windows Server:** Match your production virtual instances in a passive DR site
- **SQL Server:** One passive failover instance per licensed production instance
- **Azure Site Recovery:** Use ASR with existing SA-covered licenses for the DR target

This is especially powerful with the Arc SA tier's built-in **Disaster Recovery** capability — you get both the license rights and the Azure management tooling.

## Tracking SA Benefits with Azure Arc

Use Azure Resource Graph to audit SA and AHB utilization across your estate:

```bash
# Find all Arc-enabled servers and their license profiles
az graph query -q "
  resources
  | where type =~ 'microsoft.hybridcompute/machines'
  | extend licenseType = properties.licenseProfile.licenseType
  | project name, location, licenseType, resourceGroup
  | order by name asc
"

# Count AHB adoption across Azure VMs
az graph query -q "
  resources
  | where type =~ 'microsoft.compute/virtualmachines'
  | summarize total=count(), ahb=countif(properties.licenseType =~ 'Windows_Server')
  | extend adoption_pct = round(100.0 * ahb / total, 1)
"
```

## Getting Started

1. **Verify SA coverage** — Check the [Volume Licensing Service Center](https://www.microsoft.com/licensing/servicecenter) or ask your Licensing Solution Provider
2. **Onboard to Azure Arc** — It's free. Every server you onboard gets Basic Arc capabilities immediately
3. **Activate WS management** — For SA-covered Windows Servers, enable the "Windows Server management enabled by Azure Arc" tier in the portal
4. **Enable AHB everywhere** — Audit every Azure VM and SQL instance for AHB eligibility
5. **Enroll ESU** — If you have end-of-support servers with SA, enroll through Arc for free ESU
6. **Track with Resource Graph** — Build dashboards to monitor SA benefit utilization and identify gaps
